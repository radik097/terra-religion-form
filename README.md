# Terra Srezannaya — Religion Package Builder

Статический GitHub Pages проект для создания минимального пакета новой религии в лоре Терры Срезанной.

Проект работает без backend. Все записи сохраняются в браузере через `localStorage`. ZIP-пакет создаётся прямо в браузере через встроенный JavaScript ZIP-builder без внешних библиотек.

## Что делает сайт

1. Показывает красивую форму создания религии.
2. Объясняет простыми словами, что требуется от пользователя на каждом этапе.
3. Принимает два изображения:
   - фото значка на карте;
   - фото города.
4. Сохраняет запись в локальный реестр браузера.
5. Показывает список созданных религий.
6. Позволяет скачать ZIP-пакет отдельной религии.
7. Позволяет экспортировать и импортировать реестр в JSON.

## Минимальный ZIP-пакет религии

Каждый ZIP содержит:

```text
<religion-slug>/
├─ index.json
├─ RELIGION.md
└─ assets/
   ├─ map_icon.<png|jpg|webp|svg>
   └─ city_photo.<png|jpg|webp|svg>
```

## Поля формы

```text
Название религии
Имя автора или псевдоним
Основа религии
Описание религии
Действующие правители
Статус канона
Короткий тег
Фото значка на карте
Фото города
```

## Что требуется от пользователя

### 1. Название религии

Короткое понятное название. Оно используется в карточке и имени папки внутри ZIP.

### 2. Имя автора или псевдоним

Нужно, чтобы летописец понимал, кто предложил религию.

### 3. Основа религии

Одно предложение: на чём держится вера. Например: кровь, семя, закон, маска, вода, машина, память.

### 4. Описание религии

Одно-два предложения: кто верит, зачем верит, какую роль религия играет в мире.

### 5. Действующие правители

Кто управляет религией: правитель, совет, орден, матроны, жрецы, архивариусы, военный синод.

### 6. Фото значка на карте

Изображение, которое можно использовать как символ на карте мира. Лучше квадратное: `512x512`, PNG/WebP/SVG.

### 7. Фото города

Изображение столицы, священного города или главного поселения. Лучше широкое: `16:9`, PNG/JPG/WebP.

### 8. Скачать ZIP

После сохранения появляется карточка религии. В ней нужно нажать `Скачать ZIP`.

## Важное ограничение GitHub Pages

GitHub Pages не умеет сам сохранять общий список всех пользователей, потому что это статический хостинг.

Этот MVP сохраняет данные так:

```text
Пользователь заполнил форму → запись сохранилась в его браузере → пользователь скачал ZIP → отправил ZIP летописцу или администратору.
```

Для общего публичного реестра позже нужен backend или внешнее хранилище:

```text
GitHub Issues / Pull Requests
Supabase
Firebase
Discord Bot collector
Custom API
```

## Локальный запуск

Открой `index.html` в браузере.

Или запусти простой локальный сервер:

```bash
python -m http.server 8080
```

Потом открой:

```text
http://localhost:8080
```

## GitHub Pages deploy

1. Создай репозиторий на GitHub.
2. Загрузи файлы проекта в корень репозитория.
3. Открой `Settings → Pages`.
4. Source: `GitHub Actions`.
5. Workflow `.github/workflows/pages.yml` опубликует сайт.

## Ограничение размера изображений

Для стабильного сохранения в браузере установлено ограничение: до `3 MB` на одно изображение.

Если нужно хранить большие изображения, следующий этап — IndexedDB или backend-хранилище.
